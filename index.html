Architecture Overview

Click any component to learn how it works inside each architecture.

CPU — General Purpose
⬡ Processing Cores (×8–32)
click ↓
Each core is a fully independent processor. It can run any instruction — arithmetic, branches, memory access, I/O. Cores execute instructions sequentially within a single thread, using out-of-order execution to maximise throughput. With hyperthreading, two threads share one physical core.
◫ Cache Hierarchy L1→L2→L3
click ↓
Caches sit between cores and RAM. L1 (32KB, ~4 cycles), L2 (256KB, ~12 cycles), L3 (32MB, ~40 cycles). A cache miss stalls the pipeline until data arrives from RAM (~200 cycles). The entire memory hierarchy exists because RAM is too slow to feed the CPU directly.
⇉ Instruction Pipeline (20-stage)
click ↓
Instructions flow through Fetch → Decode → Execute → Writeback. Out-of-order execution lets the CPU re-arrange instructions to avoid stalls. Branch prediction speculatively executes the likely next path — a misprediction flushes the pipeline, costing ~15 cycles.
▦ System RAM (DDR5, ~51 GB/s)
click ↓
Main memory holds all data and instructions. DDR5 offers ~51 GB/s bandwidth, but with ~70ns latency. For matrix-heavy workloads, RAM bandwidth becomes the bottleneck — the CPU spends more time waiting for data than computing.

TPU v5e — Matrix Specialist
⊞ Matrix Multiply Unit (MXU)
click ↓
The MXU is a 128×128 systolic array of multiplier-accumulator units. It performs 128×128 matrix multiply in a single clock pulse. Each multiply-accumulate (MAC) unit does exactly one operation: multiply two numbers and add to a running total. This simplicity is why TPUs are so efficient — no branch logic, no cache management.
⊡ Systolic Array Architecture
click ↓
Data flows between PEs like a heartbeat — rhythmically, without a control unit. Each PE receives input from its top neighbour, multiplies by a stored weight, adds to the accumulator, and passes the result to its bottom neighbour. Weights flow right, activations flow down. Every PE is busy every clock cycle.
◧ High Bandwidth Memory (HBM, ~819 GB/s)
click ↓
HBM provides 16× more bandwidth than CPU DDR5. It uses 3D-stacked DRAM directly bonded to the TPU die — extremely short signal paths give ~819 GB/s bandwidth. Matrix data is fed directly into the systolic array without cache hierarchies.
◈ XLA Compiler (Static Graphs)
click ↓
The XLA (Accelerated Linear Algebra) compiler fuses operations before execution. Because TPUs can't handle dynamic branches, programs must be compiled into a static computation graph. XLA can fuse a matmul + activation + normalisation into a single kernel with zero overhead between steps.
